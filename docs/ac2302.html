AC23.02

Développer une application à partir d’un cahier des charges donné, pour le Web ou les périphériques mobiles

Ce que j’ai fait

Dans le cadre de la SAE 3.02 (Moteur de recherche), j'ai programmé en Python 3 une application de recherche sémantique multi-critères répondant à un cahier des charges strict imposant la conformité aux standards de qualité professionnels PEP 8.

Pourquoi je l’ai fait

Le respect d'un cahier des charges de codage (comme la PEP 8) garantit qu'une application est modulaire, lisible et facilement maintenable par d'autres développeurs au sein d'une entreprise. L'écriture en anglais, la documentation des fonctions et le refus de valeurs fixes (hardcoded) sont indispensables en environnement de production.

Comment je l’ai fait

J'ai segmenté mon application en fonctions métier spécialisées. J'ai utilisé des expressions régulières (Regex) pour le filtrage avancé, implémenté des constantes globales nommées (comme DOCUMENTS_DIR = "documents") et inclus systématiquement des Docstrings explicites pour documenter l'usage et les arguments de chaque fonction.

Mes difficultés

La principale difficulté résidait dans l'obligation de traiter dynamiquement plusieurs extensions de fichiers sans alourdir le code. J'ai configuré une boucle itérative analysant l'extension (file_ext) pour aiguiller proprement les flux vers les bonnes bibliothèques de lecture.

Ce que j’en ai appris

J'ai appris à structurer un projet applicatif rigoureux, à automatiser le nettoyage des entrées utilisateurs (méthode clean_query) et à intégrer des standards de codage industriels exigés en ingénierie logicielle.

# ==========================================
# CONSTANTS
# ==========================================
DOCUMENTS_DIR = "documents"

def clean_query(query_string):
    """
    Cleans the user input by replacing commas with spaces 
    and returning a list of lowercase keywords.
    """
    temp_query = query_string.replace(',', ' ')
    return [k.lower().strip() for k in temp_query.split() if k.strip()]
