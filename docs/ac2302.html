AC23.02

Développer une application à partir d’un cahier des charges donné, pour le Web ou les périphériques mobiles

Ce que j’ai fait

J'ai programmé une application web dynamique dotée d'une interface responsive, respectant les fonctionnalités exigées par le client.

Pourquoi je l’ai fait

Pour mettre à disposition des utilisateurs un outil ergonomique capable de traiter des données métiers en temps réel sans nécessiter d'installation locale.

Comment je l’ai fait

J'ai développé le frontend en **HTML5, CSS3 (framework Tailwind)** et **JavaScript**, assurant la compatibilité PC/smartphone. Le backend a été réalisé en PHP pour orchestrer le traitement des requêtes formulaires.

Mes difficultés

Garantir un affichage impeccable sur tous les navigateurs (notamment Safari mobile) a demandé de nombreux ajustements CSS sur la gestion des conteneurs flexibles (Flexbox/Grid).

Ce que j’en ai appris

J'ai appris à organiser mon code selon une logique modulaire et compris l'importance de l'expérience utilisateur (UX) dans l'adoption d'une application.

Ce que je ferais autrement

Je séparerais plus nettement le code en adoptant une architecture d'API (avec un framework moderne comme FastAPI ou Express.js) couplée à un frontend indépendant.
