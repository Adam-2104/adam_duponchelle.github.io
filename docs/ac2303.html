AC23.03

Utiliser un protocole réseau pour programmer une application client/serveur

Ce que j’ai fait

J'ai développé une application réseau bi-partie capable d'initier des connexions et d'échanger des messages bruts à travers le réseau en utilisant la couche de transport.

Pourquoi je l’ai fait

Pour comprendre comment les applications comme SSH, HTTP ou les jeux en ligne communiquent réellement avec les cartes réseaux sous-jacentes du système.

Comment je l’ai fait

J'ai utilisé l'API **Sockets** de Python. J'ai codé un script serveur écoutant sur un port TCP spécifique, capable de gérer les connexions entrantes, et un script client envoyant des chaînes de caractères encodées en UTF-8 avant de recevoir un accusé de réception.

Mes difficultés

Au début, mon serveur se bloquait dès qu'un premier client se connectait, interdisant l'accès aux autres. J'ai dû implémenter le multi-threading (module threading) pour que le serveur crée un fil d'exécution dédié à chaque nouveau client.

Ce que j’en ai appris

J'ai compris le mécanisme du "Three-way handshake" de TCP, la notion de port d'écoute et la gestion des flux d'octets asynchrones.

Ce que je ferais autrement

J'ajouterais une couche de sécurité applicative essentielle en encapsulant le flux réseau dans un tunnel chiffré **TLS/SSL** (via le module ssl de Python) pour chiffrer les données échangées.

📂 Preuve : tp_sockets_client_serveur.py
