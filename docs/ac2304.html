AC23.04

Installer, administrer un système de gestion de données

Ce que j’ai fait

J'ai déployé un moteur de base de données relationnelle, modélisé son schéma logique et géré les privilèges d'accès.

Pourquoi je l’ai fait

Une application a besoin de stocker ses informations de manière persistante, structurée et sécurisée pour éviter les pertes de données ou les accès frauduleux.

Comment je l’ai fait

J'ai installé un serveur **MySQL / MariaDB** sous Linux. J'ai conçu le dictionnaire de données, rédigé le script SQL de création des tables (avec clés primaires et étrangères pour l'intégrité référentielle) et créé des utilisateurs restreints avec des droits spécifiques (SELECT, INSERT) via la CLI.

Mes difficultés

La mise en place des contraintes de clés étrangères (FOREIGN KEY) générait des erreurs lors des insertions initiales. J'ai dû réorganiser l'ordre d'exécution de mes scripts pour que les tables parentes soient alimentées avant les tables enfants.

Ce que j’en ai appris

J'ai appris l'importance de la normalisation des bases de données (Formes Normales) pour éliminer la redondance d'information et optimiser la rapidité des requêtes.

Ce que je ferais autrement

Je planifierais dès l'installation un script automatisé de sauvegarde (dump SQL) via une tâche Cron, déportant les sauvegardes sur un stockage distant pour assurer un plan de reprise d'activité (PRA).
