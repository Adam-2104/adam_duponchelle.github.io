AC23.05

Accéder à un ensemble de données depuis une application et/ou un site web

Ce que j’ai fait

J'ai connecté mon application web à la base de données afin de lire, modifier et afficher dynamiquement les informations sur l'interface utilisateur.

Pourquoi je l’ai fait

Un site web statique ne permet pas d'interaction. Connecter la base de données permet d'avoir des comptes utilisateurs fonctionnels, des catalogues mis à jour ou des tableaux de bord dynamiques.

Comment je l’ai fait

En PHP, j'ai utilisé l'extension **PDO (PHP Data Objects)** pour initialiser une connexion sécurisée vers le SGBD. J'ai rédigé des requêtes de sélection complexes à base de jointures (JOIN) pour récupérer les informations associées et les injecter proprement dans des boucles d'affichage HTML.

Mes difficultés

Mon application était vulnérable aux injections SQL lorsque je passais directement des variables dans mes requêtes. J'ai dû corriger tout le code en utilisant systématiquement des **requêtes préparées** (fonctions prepare() et execute()) pour neutraliser les entrées utilisateurs.

Ce que j’en ai appris

J'ai appris à lier le backend à un modèle de données persistant et j'ai assimilé les règles d'or de la sécurité applicative liée au traitement des bases de données.

Ce que je ferais autrement

J'implémenterais un **ORM (Object-Relational Mapping)** comme Eloquent ou Doctrine. Cela permet d'interagir avec la base de données directement sous forme d'objets, rendant le code beaucoup plus propre et indépendant du SGBD utilisé.
