Sécuriser les services

← Retour à la compétence

Description de la Réalisation (SAÉ 4.Cyber.01)

Afin de protéger les services serveurs vitaux exposés dans la zone démilitarisée (DMZ), notre groupe a procédé à l'endurcissement des configurations applicatives des serveurs Web et DNS.

Détails Techniques de la Preuve :

  • Sécurisation Web (HTTPS) : Configuration du serveur Web Apache pour forcer les redirections permanentes HTTP (port 80) vers HTTPS (port 443). Injection d'en-têtes HTTP de sécurité pour prémunir le serveur contre le clickjacking et le vol de cookies (HSTS, X-Frame-Options, X-XSS-Protection).
  • Sécurisation DNS (DNSSEC) : Implémentation du protocole d'extension de sécurité DNSSEC sur le serveur de noms BIND9. Signature complète de la zone DNS d'entreprise à l'aide de paires de clés de signature de zone (ZSK) et de clés de signature de clé (KSK). Validation des enregistrements cryptographiques (RRSIG, DNSKEY).
Livrable : Fichiers de zone signés & Fichiers de configuration Apache
